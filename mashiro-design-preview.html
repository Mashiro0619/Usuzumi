15 / Focus & Accessibility

Focus & Contrast Audit

Focus Ring

Rings use :focus-visible — they appear for keyboard navigation only, not mouse clicks. Composed via box-shadow: var(--focus-ring) so they follow the element's existing border-radius.

Soft-Border Note

Default / Soft borders sit below WCAG 3 : 1 intentionally (1.15 – 1.27 : 1) for the paper aesthetic. Never rely on a border alone to signal an interactive control — pair it with readable text or icon affordance that meets contrast on its own.

Accessibility Requirements

  • ColorMeaning is never carried by color alone; pair tone with text, icon, shape, or position.
  • KeyboardInteractive elements stay reachable in source order, with visible focus and Escape behavior for overlays.
  • MotionReduced-motion preferences remove animation while preserving focus visibility and state feedback.
  • Forced ColorsNative controls, visible outlines, and border-based structure remain intact in high-contrast modes.
